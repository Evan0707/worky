PARTIE 1 — ÉTAT DES LIEUX (ce qui est déjà construit)
================================================================================

Tu es beaucoup plus loin que tu ne le crois. L'app est déjà une vraie suite
de gestion de chantier, pas un MVP bancal.

BACKEND (17 routers tRPC, 23 modèles Prisma)
  • project, task, time-entry, material, photo, invoice
  • message (chat temps-réel Pusher), project-note, notification
  • tag, team + team-invitation, project-template
  • safety-checklist, planning, report
  • stripe (paiements), user (auth/profil)

FRONTEND (dashboard 100% i18n — FR/EN/DE/ES)
  Workspace chantier : photos, temps, matériaux, messages, notes, tâches,
  équipe, sécurité. Plus : planning, rapports, factures.

DIFFÉRENCIATEURS DÉJÀ EN PLACE
  1. Factur-X / ZUGFeRD / Peppol → quasi aucun concurrent français le fait
     bien pour les artisans solo.
  2. Magic link client sans compte → UX ultra-fluide pour le client final.
  3. Multi-lingue natif (rare dans la verticale BTP).
  4. PWA → pas de store Apple/Google à gérer, installable, offline-ready.
  5. Stack moderne (Next 15, tRPC, Neon, Edge) → vélocité de dev x10 vs
     concurrents sur stacks legacy.

CE QUI MANQUE ENCORE (gaps prioritaires)
  • Devis/offres (DEVIS → facture conversion) — critique pour l'artisan.
  • Signature électronique sur devis (Yousign / DocuSign / maison).
  • OCR ticket de caisse → matériau auto (photo → ligne créée).
  • Export compta (FEC, écritures CIEL/EBP/Sage).
  • Géolocalisation pointage (anti-triche temps sur chantier).
  • Mode hors-ligne PWA robuste (aujourd'hui l'app casse sans réseau).
  • Intégration bancaire (Bridge/GoCardless) pour réconciliation paiements.
  • App mobile native ? — PWA suffit 12 mois, mais Play Store/App Store
    boostent le SEO "install".


PARTIE 2 — VISION PRODUIT (où aller dans les 18 mois)
================================================================================

POSITIONNEMENT MARCHÉ
  Tu n'es PAS Batappli, PAS Obat, PAS Hellio. Ceux-là visent des gros
  artisans avec un contrôleur de gestion. Toi tu vises :

      "Le plombier / électricien / maçon solo ou 1-3 salariés, 30-50 ans,
       qui tient encore son chantier sur WhatsApp + Excel + carnet."

  Ce segment = 600 000 entreprises artisanales en France. Personne ne les
  sert correctement aujourd'hui. Les outils existants sont soit :
    - trop chers (80-150€/mois pour Batappli, Obat)
    - trop compliqués (CRM + ERP + compta mélangés)
    - laids/datés (UX années 2010)

TROIS HORIZONS

  HORIZON 1 (T2 2026 — "Compléter le cœur métier")
    → Devis PDF + signature client via magic link
    → OCR ticket de caisse (Mistral OCR ou GPT-4o vision, 2 centimes l'appel)
    → Mode offline PWA (Dexie + sync queue)
    → Export FEC basique
    Objectif : un artisan peut REMPLACER complètement son carnet + Excel.

  HORIZON 2 (T3-T4 2026 — "Cross-sell & verrouillage")
    → Intégration bancaire (Bridge API — réconciliation paiements auto)
    → Module sous-traitance (2 artisans différents facturent le même chantier)
    → Templates métier (plombier, électricien, peintre, maçon, chauffagiste)
      avec checklists sécurité et matériaux pré-remplis par métier
    → Application Play Store + App Store via PWA wrap (Bubblewrap / PWABuilder)
    Objectif : une fois dedans, l'artisan ne peut plus sortir.

  HORIZON 3 (2027 — "Plateforme & écosystème")
    → API publique + webhooks (intégrateurs, comptables, assureurs)
    → Marketplace matériaux (Leroy Merlin Pro, Cedeo, Rexel via API prix)
    → "Réseau OpenChantier" : artisans se recommandent entre eux,
      sous-traitance entre membres. Network effect = moat.
    → Expansion DE puis BE/NL (Peppol déjà prêt).


PARTIE 3 — VISION TECHNOLOGIQUE (paris à faire)
================================================================================

  IA UTILE (pas gadget) :
    - Photo chantier → génération note auto (description visuelle)
    - Ticket matériau → parsing auto (déjà mentionné)
    - Génération devis à partir de 3 bullet points
    - Résumé de conversation client (semaine = 1 paragraphe)

  PLATEFORME :
    - Ne pas partir sur React Native. La PWA couvre 95% des besoins.
    - Investir dans le mode offline AVANT d'investir dans du natif.
    - Pousser fort sur la WebShare API (partager photo chantier) + Media
      Capture (photo directe depuis le navigateur).

  MONITORING PRODUIT :
    - PostHog (gratuit jusqu'à 1M events) pour funnel + feature flags
    - Plausible Analytics pour marketing (RGPD friendly, pas de bandeau
      cookie nécessaire)
    - Sentry déjà en place.

PARTIE 7 — RISQUES & ANTI-PATTERNS
================================================================================

  À ÉVITER
  ✗ Dépenser en Ads avant d'avoir un funnel qui convertit >3%
  ✗ Construire des features avant de parler aux utilisateurs
     (fais 20 interviews clients par mois — obligatoire)
  ✗ Faire du "mode entreprise" (SSO, rôles complexes) avant 1000 clients
  ✗ Sous-estimer l'UX mobile — l'artisan va utiliser à 80% sur téléphone
  ✗ Copier Batappli/Obat dans leurs fonctionnalités — tu perdras.
     Différencie-toi par la simplicité, le prix, l'UX.

  À SURVEILLER
  ⚠ Concurrents qui copient l'UX dès qu'ils te voient monter
     → ta vitesse d'itération = ton seul vrai moat
  ⚠ Réglementation facture électronique française (dates qui bougent)
  ⚠ Stripe fees sur les petites transactions (15€/mois) — checker la
     rentabilité unitaire
