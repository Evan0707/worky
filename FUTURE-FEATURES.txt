--------------------------------------------------------------------------------

Constat :
  Aujourd'hui OpenChantier gere la facture (Invoice) mais PAS le devis. Or
  le workflow artisan est : prospect -> devis -> signature -> acompte ->
  chantier -> factures intermediaires -> facture finale. Sans devis, on
  attaque le marche par le milieu.

2. SIGNATURE ELECTRONIQUE
--------------------------------------------------------------------------------

Constat :
  Pour passer d'un devis a un chantier, il faut la signature du client. Aujourd
  'hui les artisans impriment le devis, font signer, scannent. OpenChantier
  peut remplacer ca par une signature dans le magic link.

Niveaux de signature (eIDAS) :
  - SES (Simple) : case a cocher + IP + user-agent + timestamp. Valeur legale
    pour contrats < 10 000 EUR en general. C'est le MVP.
  - AES (Avancee) : envoi SMS OTP + certificat a usage unique. Requis pour
    devis > 10 000 EUR ou contentieux probables. Via DocuSign / Yousign API.
  - QES (Qualifiee) : equivalent manuscrit. Hors scope court terme.

Implementation MVP (SES) :
  - Dans magic link, bouton "Accepter le devis"
  - Modal : afficher prix, conditions, ouverture pad de signature (SignaturePad
    react)
  - Stocker : signature base64 + IP + user-agent + timestamp + hash SHA-256 du
    PDF au moment de la signature (horodatage RFC 3161 via freeTSA.org ou
    universign)
  - Genere un certificat de signature PDF attache au devis signe
  - Email de confirmation automatique artisan + client avec PDF signe

Plan : PRO.

3. RELANCES AUTOMATIQUES IMPAYES
--------------------------------------------------------------------------------

Constat :
  Les artisans perdent 5-15% de leur CA en impayes. Les relances manuelles sont
  chronophages et souvent oubliees. Automatiser ca a une valeur enorme.

Scenario :
  Par facture emise, 3 relances automatiques :
    - J+3 apres echeance : email courtois ("petit rappel amical")
    - J+15 : email ferme + SMS (Twilio / OVH SMS)
    - J+30 : mise en demeure recommandee digitale (AR24 / Docaposte) OU
      PDF a signer par l'artisan et envoyer en LRAR papier

Config par artisan :
  - Delais personnalisables
  - Templates de relance multilingues (auto-traduit selon locale client)
  - Toggle ON/OFF par facture
  - Penalites de retard calculees automatiquement (3x taux legal +
    indemnite forfaitaire 40 EUR pour pro-a-pro, art. L.441-10 code commerce)

Schema additions :

  model InvoiceReminder {
    id          String     @id @default(cuid())
    invoiceId   String
    stage       Int        // 1, 2, 3
    sentAt      DateTime
    channel     String     // EMAIL, SMS, LRAR
    opened      Boolean    @default(false)
    invoice     Invoice    @relation(fields: [invoiceId], references: [id], onDelete: Cascade)
  }

Technique :
  - Cron Vercel (vercel.json) ou QStash : run toutes les 6h
  - Hook sur Invoice.paidAt pour arreter la sequence
  - Dashboard : widget "A relancer" sur home

Plan : PRO.

4. OCR TICKETS & FACTURES FOURNISSEURS
--------------------------------------------------------------------------------

Constat :
  Un artisan passe 2-5h/semaine a saisir ses achats. Scanner ticket Point.P,
  relever : fournisseur, date, montant HT, TVA, articles. L'OCR + IA est mature
  en 2026 et cette feature se vend tres bien.

Stack technique :
  - Photo prise dans l'app (deja existant via UploadThing)
  - Envoi a un microservice Python avec :
      * Tesseract pour OCR brut
      * Claude Haiku / GPT-4o-mini pour extraction structuree (JSON)
  - Retour : { vendor, date, totalHt, vat, items[{description, qty, price}] }
  - Pre-remplissage du formulaire Material (user valide)

Alternatives commerciales :
  - Mindee OCR (API francaise, modele recu specialisee, 0.10 EUR/scan)
  - Klippa (0.08 EUR/scan)
  - AWS Textract + Bedrock (plus technique, moins cher au volume)

Quotas :
  - FREE : 5 scans/mois
  - PRO : 100 scans/mois inclus, au-dela 0.20 EUR/scan

Plan : PRO (FREE teaser 5/mois pour conversion).

5. MODE HORS-LIGNE (PWA OFFLINE-FIRST)
--------------------------------------------------------------------------------

Constat :
  Sur chantier : cave, sous-sol, zone blanche, rase campagne = pas de reseau.
  Si l'app ne marche pas hors-ligne, c'est un deal-breaker. C'est l'objection
  n1 des artisans de terrain.

Architecture :
  - Service Worker (next-pwa) : cache shell + assets
  - IndexedDB (Dexie.js) : cache des projets recents + time entries non
    sync + photos en attente d'upload
  - Queue de mutations : tRPC calls stockes offline, rejoues a la reconnexion
  - Resolution de conflits : "last write wins" pour notes, "merge" pour
    time entries (jamais perdre un pointage)
  - Indicateur UI : banniere "Mode hors ligne - X modifications en attente"

Scope offline (MVP) :
  - Lecture : projets actifs, photos deja chargees, notes, taches
  - Ecriture : pointer temps, prendre une photo (upload differe), ajouter
    une note, cocher une tache, scanner un materiau

Scope exclu offline :
  - Creation nouveau projet (pas critique)
  - Facturation / devis (necessite verif PRO + numero sequentiel)
  - Settings

Plan : FREE + PRO (c'est un fondamental du produit).

avec reauth SCA, stockage chiffre des tokens.

10. EXPORT COMPTABLE (FEC, CIEL, EBP, SAGE)
--------------------------------------------------------------------------------

Tout expert-comptable demande un export. Sans ca, l'artisan doit ressaisir.

Formats :
  - FEC (Fichier des Ecritures Comptables) : norme DGFiP obligatoire, TXT

  - Ciel Compta : XIMPORT.TXT
  - EBP Compta : CSV format EBP
  - Sage : PNM (via Sage 50 Cloud API)
  - CSV generique + Pennylane / Indy / Dougs API (cabinets modernes)

UX : settings > comptabilite > choisir format > export mensuel auto envoye
a l'expert-comptable par email.

Plan : PRO.

11. TVA MULTI-TAUX & AUTOLIQUIDATION BTP
--------------------------------------------------------------------------------

Cas complexes a gerer correctement :
  - Neuf : 20%
  - Amelioration maison +2 ans : 10%
  - Renovation energetique eligible : 5,5% (avec attestation particulier)
  - Monuments historiques : 10%
  - Franchise en base (micro-entreprise < 36 800 EUR) : pas de TVA, mention
    "TVA non applicable art. 293 B du CGI"
  - Autoliquidation BTP sous-traitance : mention "Autoliquidation" sur
    facture, pas de TVA facturee
  - Intracommunautaire B2B : mention "Exoneration TVA art. 262 ter I CGI"

UX : formulaire de facture detecte le contexte et propose le bon regime,
avec info-bulle expliquant pourquoi. Attestations PDF automatiques pour
TVA reduite.

Plan : FREE (standards) | PRO (cas complexes + attestations auto).

14. POINTAGE GEOLOCALISE
--------------------------------------------------------------------------------

Pour les patrons de 2-10 salaries :
  - Le salarie pointe arrivee/depart : l'app capture GPS
  - Verifie qu'il est bien dans un rayon de 200m du chantier
  - Si non : alerte patron + demande justification
  - RGPD : consentement explicite, desactivation possible par salarie,
    geolocalisation UNIQUEMENT au moment du pointage (pas de tracking continu)

Conformite :
  - CNIL : geolocalisation salaries reglementee, declaration RGPD obligatoire
  - Convention collective BTP : temps de trajet a prendre en compte
  - Code du travail : droit a la deconnexion, pas de pointage apres 20h

Plan : PRO uniquement (feature pro). Option activable par compte.

================================================================================
  CRITERES DE PRIORISATION
================================================================================

Chaque feature a ete evaluee sur :
  - Valeur utilisateur (pain actuel * frequence)
  - Willingness to pay (converti PRO ?)
  - Cout de dev (en semaines)
  - Risque reglementaire (retard = contentieux)
  - Differentiation vs concurrents (Evoliz, Tolteck, Obat, Clicfacture,
    Archipad, Kizeo)

Regle d'or : une feature P0 manquante empeche un artisan de remplacer
son outil actuel. Une feature P3 est un +.

Tolteck          : devis/factures mobile, pas de photos ni planning -> on
                   gagne sur polyvalence
Obat             : BTP specifique, tres complet mais UX desuete -> on gagne
                   sur design + mobile PWA
Evoliz           : compta+facturation, pas specialise BTP -> on gagne sur
                   features metier (safety, planning, photos)
Clicfacture      : micro-BTP simple -> on gagne sur completude

                   sur fonctionnalites lourdes, on gagne sur prix + web
Kizeo Forms      : formulaires custom -> parallele, pas vrai concurrent
Batappli         : logiciel desktop vieillissant -> on gagne sur tout
Mediabat / EBP   : suite desktop complete -> on gagne sur prix + web
Houzz Pro (US)   : marche US, peu implante FR -> a surveiller si entree EU

Positionnement OpenChantier : le meilleur rapport polyvalence / prix /
UX moderne pour artisans solo et TPE (< 10 salaries). Focus metiers du
batiment second oeuvre (plomberie, electricite, peinture, menuiserie,
carrelage, platrerie) avant gros oeuvre.
